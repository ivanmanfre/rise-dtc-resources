Original brief · Emil

Import all new users who sign up to the platform into HubSpot and Airtable, on an ongoing basis.

Set up a webhook that sends new signup data to our developers, who'll forward it into the Airtable / n8n setup.

Run the same check against existing customers and contacts in HubSpot, so we can tell whether a new signup is already a customer, lead, or contact.

In short:

  1. A webhook for all new platform signups
  2. The relevant user / company data included in the webhook
  3. Match new signups against existing HubSpot records
  4. Data sent into the Airtable / n8n workflow
What I sent back · Iván

Happy to help on this. I'll set up an n8n webhook endpoint and send the URL to you or your devs, so they can POST signup data to it whenever a new user signs up.

Two things to clarify before I quote.

1. Match behavior

When we find an existing HubSpot record (contact / lead / customer), should we:

  1. Just tag the Airtable row with the matched record link,
  2. Also update the HubSpot record (e.g., set a "Signed up to platform" property),
  3. Or also create a new contact in HubSpot if there's no match?
2. Slack notification

Do you want a Slack ping for new signups, same as the Leads flow? If yes, same channel or a dedicated #signups one? Anything you want highlighted in the message (matched HubSpot status, plan signed up for, etc.)?
